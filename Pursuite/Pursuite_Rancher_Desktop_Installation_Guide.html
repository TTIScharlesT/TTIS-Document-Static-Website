Introduction

Purpose

This guide explains how to install Pursuite 1.0 standalone on Windows when Rancher Desktop is used as the local container runtime instead of Docker Desktop. It covers Rancher Desktop prerequisites, required engine settings, Pursuite installer and portable launcher procedures, first-run access, validation, rollback, and troubleshooting.

Scope

This guide applies to Pursuite standalone RC QA and pilot environments that intentionally use Rancher Desktop for local container management. The current Pursuite Manager startup path uses Docker-compatible commands and Docker Compose, so Rancher Desktop must expose the Docker CLI and compose workflow.

Intended Audience

The primary audience is administrators, support engineers, release QA testers, and technically capable SMB pilot users who install Pursuite on Windows with Rancher Desktop.

Document Conventions

  • Procedures use numbered steps.
  • File paths use Windows path syntax.
  • Commands are written for PowerShell unless otherwise noted.
  • Values marked Information Not Yet Available require confirmation before final publication.
  • Rancher Desktop references are based on official Rancher Desktop documentation current on June 22, 2026.

Rancher Desktop Configuration Requirements

Required Engine Mode

Configure Rancher Desktop to use Moby/dockerd, not containerd-only mode.

Reason: Pursuite Manager currently starts the stack by calling Docker-compatible commands, including image loading and Docker Compose. Rancher Desktop provides Docker CLI behavior when Moby/dockerd is selected. In containerd mode, Rancher Desktop uses nerdctl for container management, which is not the current Pursuite Manager startup path.

Required Command Verification

After Rancher Desktop starts, open PowerShell and run:

docker version
docker compose version
docker ps

Expected result:

  • docker version returns client and server information.
  • docker compose version returns a compose version.
  • docker ps returns an empty or populated container list without a daemon connection error.

Privileged Service Consideration

Rancher Desktop documentation notes that the privileged service is required to expose container services on all host interfaces. Install Rancher Desktop in a mode that supports the Pursuite web port binding required by the local stack. If the privileged service is not installed, port exposure may be limited and Pursuite startup may fail until the port configuration is adjusted or the service is installed.

Kubernetes Setting

Pursuite standalone does not require Kubernetes for baseline operation. If Rancher Desktop allows Kubernetes to be disabled while preserving Moby/dockerd and Docker Compose functionality, disabling Kubernetes may reduce resource usage. Validate this in the target environment before documenting it as a required setting.

Installation

Overview

Pursuite can be installed in two ways after Rancher Desktop is configured:

  1. Installer: Pursuite-Standalone-Setup-1.0.0.exe
  2. Portable launcher: Pursuite-Standalone-Launcher-1.0.0.exe

For RC QA, artifact names may include an rc suffix. RC artifacts are unsigned unless explicitly signed for testing.

Install Rancher Desktop

  1. Confirm Windows Subsystem for Linux is installed.
  2. Download the Rancher Desktop Windows installer from the official Rancher Desktop releases page.
  3. Run the Rancher.Desktop.Setup.X.Y.Z.msi installer.
  4. Review and accept the license agreement.
  5. If prompted, choose the installation scope required by your organization.
  6. Prefer an installation mode that includes the Rancher Desktop privileged service when Pursuite needs host port exposure.
  7. Finish the installation wizard.
  8. Launch Rancher Desktop.
  9. Configure the container engine as Moby/dockerd.
  10. Wait until Rancher Desktop reports ready.

Expected result: PowerShell can run docker version, docker compose version, and docker ps successfully.

Verify The Pursuite Artifact

Use the release manifest when available. For RC QA, compare the provided SHA256 hash.

Get-FileHash "Pursuite-Standalone-Setup-1.0.0.exe" -Algorithm SHA256
Get-FileHash "Pursuite-Standalone-Launcher-1.0.0.exe" -Algorithm SHA256

Expected result: the displayed hash matches the hash published for the build under test.

Install With The Windows Installer

  1. Confirm Rancher Desktop is running and configured for Moby/dockerd.
  2. Confirm Docker-compatible commands work from PowerShell.
  3. Locate the Pursuite installer artifact.
  4. Right-click the installer and select Run as administrator if local policy requires elevated installation.
  5. Follow the installer prompts.
  6. Select the installation directory if prompted.
  7. Finish the installer.
  8. Start Pursuite from the Start Menu or desktop shortcut.

Expected result: Pursuite Manager opens or starts in the tray, prepares the local environment, starts the stack when requested, and opens the local Pursuite web app after services become healthy.

Run The Portable Launcher

  1. Confirm Rancher Desktop is running and configured for Moby/dockerd.
  2. Copy the portable launcher to a local folder.
  3. Double-click the launcher.
  4. If Windows warns that the app is unsigned during RC QA, confirm only if you intentionally trust the RC build.
  5. Wait for Pursuite Manager to open.
  6. Click Launch Pursuite if the stack does not start automatically.

Expected result: Pursuite Manager starts the local stack through Rancher Desktop and opens the local Pursuite web app.

First-Run Admin Access

On a clean standalone install, Pursuite Manager creates a first-run credential file after the stack becomes healthy.

Path:

%APPDATA%\Pursuite\first-run-admin.txt

Use:

  • Organization: default
  • Email: admin@pursuite.local
  • Temporary password: value shown in first-run-admin.txt

After first sign-in:

  1. Create a named admin user.
  2. Confirm the named admin can sign in.
  3. Rotate or clear the temporary local admin password.
  4. Store installation evidence in the pilot QA record.

Validate Installation

  1. Open Pursuite Manager.
  2. Confirm the Dashboard shows version v1.0.0.
  3. Confirm the services show healthy or running status:
    • PostgreSQL
    • Redis
    • API Server
    • Web / nginx
  4. Click Open App.
  5. Sign in with the first-run admin account.
  6. Create or confirm a backup admin account.
  7. Create a test account, contact, opportunity, and proposal.
  8. Run a backup from Manager -> Backups.

Expected result: the local stack is healthy, login works, core objects can be created, and a backup completes.

Rollback Procedure

If installation fails:

  1. Close Pursuite Manager.
  2. Stop the Pursuite stack from Manager if available.
  3. Preserve %APPDATA%\Pursuite for troubleshooting.
  4. Export or copy launcher.log.
  5. Do not delete container volumes unless data removal is intentional and approved.
  6. Reinstall the prior working RC or build if required.

Uninstall Procedure

  1. Use Windows Apps & features or the Start Menu uninstaller if installed through the setup executable.
  2. Confirm whether application data should be preserved.
  3. Do not remove container volumes unless explicitly requested.
  4. Preserve backups before removing local data.

Expected result: the application is removed, and local proposal data remains available unless intentionally removed.

Uninstall Rancher Desktop

Uninstall Rancher Desktop only after confirming that no Pursuite data or local container volumes are required. Removing or resetting Rancher Desktop may remove local runtime state depending on the selected options.

Usage Procedures

Start Pursuite

Purpose

Start the local container stack and open the Pursuite web application.

When To Use

Use this procedure after installation, reboot, Rancher Desktop restart, or stack shutdown.

Step-By-Step Instructions

  1. Start Rancher Desktop.
  2. Wait until Rancher Desktop reports ready.
  3. Open PowerShell and confirm docker ps succeeds.
  4. Open Pursuite Manager.
  5. Click Launch Pursuite.
  6. Wait for services to become healthy.
  7. Click Open App if the browser does not open automatically.

Expected Results

The local Pursuite web app opens and the Manager shows healthy services.

Verification

Confirm all services are healthy in Manager and sign in successfully.

Troubleshooting

If services do not start, open Manager -> Logs, review launcher.log, and confirm Rancher Desktop is still using Moby/dockerd.

Stop Pursuite

  1. Open Pursuite Manager.
  2. Click Stop Stack.
  3. Wait for services to stop.

Expected result: the stack stops without deleting local volumes.

Run A Backup

  1. Open Pursuite Manager.
  2. Select Backups.
  3. Click the backup action.
  4. Wait for completion.
  5. Confirm the backup file appears in the backup directory.

Expected result: backup completes and is available under %APPDATA%\Pursuite\backups.

Restore A Backup

  1. Open Pursuite Manager.
  2. Select Backups.
  3. Choose the restore action.
  4. Select the backup file.
  5. Confirm the restore.
  6. Restart the stack.

Expected result: users, proposals, uploads, and content from the backup are available after restart.

Collect Diagnostics

  1. Open Pursuite Manager.
  2. Open the Dashboard.
  3. Click Copy Diagnostics.
  4. Save the copied text with the QA issue or support ticket.
  5. Attach %APPDATA%\Pursuite\launcher.log when escalation is required.
  6. Include Rancher Desktop version and engine mode in the ticket.

Expected result: support receives version, paths, runtime setting, service status, and environment details.

Troubleshooting

Issue Cause Resolution
Docker command fails Rancher Desktop is not running, not ready, or not configured for Moby/dockerd. Start Rancher Desktop, switch to Moby/dockerd, and retry docker ps.
Compose command fails Docker Compose is unavailable or Rancher Desktop engine mode is incompatible. Confirm docker compose version works from PowerShell.
Stack does not start Runtime unavailable, image load failure, port conflict, or compose error. Open Manager logs, review launcher.log, confirm Rancher Desktop readiness, and retry.
Port conflict Another service is using the configured web port. Stop the other service or change HTTP_PORT in .env.production.
Port exposure fails Rancher Desktop privileged service is missing or host port exposure is restricted. Reinstall or repair Rancher Desktop with privileged service support, or adjust the Pursuite port configuration.
Browser shows stale UI Browser cache or service worker retained older assets. Hard refresh the browser and clear local site data if required.
First-run admin file missing Stack has not become healthy or an admin already exists. Wait for stack health, then check %APPDATA%\Pursuite\launcher.log.
Login fails Wrong organization, email, or temporary password. Use organization default, email admin@pursuite.local, and the generated credential file.
Containers unhealthy API, database, Redis, or web service failed health checks. Review Manager logs and Rancher Desktop container logs.
Backup fails Stack is unhealthy, storage unavailable, or runtime operation failed. Confirm services are running and retry from Manager.
Restore fails Invalid backup, stack conflict, or storage issue. Preserve logs, restart stack, and retry with a known-good backup.
Installer warning appears RC build is unsigned or SmartScreen has not established reputation. For QA, proceed only with trusted RC artifacts. For release, use signed binaries only.

Frequently Asked Questions

Can Pursuite run with Rancher Desktop?

Yes for QA when Rancher Desktop is configured with Moby/dockerd and Docker Compose works from PowerShell. Final release support status must be validated before publication.

Can Pursuite run with Rancher Desktop containerd mode?

Not with the current Pursuite Manager startup path. Pursuite Manager expects Docker-compatible commands and Docker Compose.

Is Kubernetes required?

No. Pursuite standalone does not require Kubernetes for baseline operation. Validate the final Rancher Desktop configuration before disabling Kubernetes in a pilot environment.

Can I use the portable launcher instead of the installer?

Yes. The portable launcher is a first-class RC and release artifact for standalone validation.

Where is local data stored?

Configuration and logs are stored under %APPDATA%\Pursuite. Application data and uploads are stored in local container volumes managed by the runtime.

Does uninstall delete proposal data?

Uninstall must not silently delete proposal data. Container volumes should be preserved unless data removal is explicitly requested.

Why does Windows show a warning for RC builds?

RC artifacts may be unsigned while the code-signing certificate is pending. Production release artifacts must be signed.

  • Disable optional runtime features only after validation.

Security Recommendations

  • Rotate first-run credentials.
  • Restrict access to backup files and local environment files.
  • Restrict access to Rancher Desktop runtime controls.

Maintenance Guidance

  • Keep backup evidence with release QA records.

  • Validate reinstall-over-existing-data before release.
  • Revalidate Pursuite after Rancher Desktop upgrades.
