Introduction

Purpose

The Pursuite RFP Shredder helps proposal teams convert solicitation documents into a structured, editable requirements list. Confirming the requirements unlocks downstream proposal workflows, including the Compliance Matrix and the Autonomous Pipeline.

Scope

This guide covers:

  • Uploading an RFP, RFI, RFQ, SOW, or related solicitation document into the RFP Shredder when that document is represented as a supported RFP upload.
  • Reviewing, editing, adding, deleting, and confirming extracted requirements.
  • Using the Proposal Kickoff wizard when available.
  • Generating and reviewing the Compliance Matrix after requirements are confirmed.
  • Troubleshooting verified upload, processing, review, and confirmation issues.

This guide does not document solicitation content strategy, proposal writing policy, or organization-specific capture methodology.

Intended Audience

Audience Typical Use
Proposal Manager Upload and confirm RFP requirements, manage proposal readiness gates, and generate the Compliance Matrix.
Admin Assist Proposal Managers, resolve blocked workflows, and configure prerequisite AI or CUI controls.
Writer Review completed requirements and use them as proposal context.
Executive Approver View requirements and downstream proposal context where access is permitted.
Read-Only user [SME Review Required: RFP Shredder visibility for Read-Only users is not verified for this guide.]

Benefits

  • Reduces manual transcription of solicitation requirements.
  • Creates a structured requirement set for review.
  • Identifies mandatory and optional requirements.
  • Supports proposal compliance review through the Compliance Matrix.
  • Preserves review and confirmation gates before downstream automation runs.

Key Terminology

Understanding The Shredding Process

What Is An RFP Shred?

An RFP shred is the process of turning solicitation text into a structured list of requirements that the proposal team can review, edit, confirm, and use for compliance tracking.

Why Shredding Is Performed

Proposal teams shred RFPs to:

  1. Identify what the solicitation requires.
  2. Separate mandatory requirements from optional or contextual items.
  3. Preserve a reviewed requirement baseline.
  4. Map requirements to proposal sections.
  5. Detect missing or partial proposal coverage before final review.

Verified Shred Workflow

Step System Behavior
Upload Admin or Proposal Manager uploads a PDF or DOCX file up to 50 MB.
Queue The file is queued for analysis.
Extract AI extracts requirements. PDF files can use OCR fallback when text extraction returns no text.
Review Authorized users edit, add, delete, or mark requirements reviewed.
Confirm Admin or Proposal Manager confirms the reviewed set.
Unlock Compliance Matrix and Autonomous Pipeline become available for the proposal.

Step-By-Step: Upload A Solicitation Document

  1. Open the proposal workspace.
  2. Open RFP Shredder.
  3. If no document has been uploaded, locate the upload area that says Drop your RFP here or browse.
  4. Drop a supported file onto the upload area, or select browse.
  5. Choose a PDF or DOCX file.
  6. Confirm the file is 50 MB or smaller.
  7. Wait while the upload completes.
  8. Watch the processing status.

[Screenshot: RFP Shredder upload area]

Expected result: The document is accepted and processing starts. If the document is unsupported or too large, Pursuite shows an error.

Supported File Types

Format Verified Support Notes
PDF Yes Text extraction is used first. OCR fallback is available for scanned or image-only PDFs.
DOCX Yes Word document text extraction is used. OCR is not used for DOCX.
Other formats No verified support [SME Review Required: Confirm whether additional file types are planned.]

Step-By-Step: Review Extracted Requirements

  1. Wait until the RFP Shredder status is Completed.
  2. Review the extracted requirement count.
  3. Review the mandatory and optional counts.
  4. Review each requirement row.
  5. Compare low-confidence rows to the source solicitation.
  6. Look for rows marked Review required.
  7. Confirm whether each row belongs in the final requirement set.

[Screenshot: Extracted requirements list with category badges]

Expected result: You have identified incorrect, missing, duplicate, or low-confidence requirements before confirmation.

Step-By-Step: Edit A Requirement

  1. Hover over the requirement row.
  2. Select the edit control.
  3. Update Requirement text as needed.
  4. Select the correct Category.
  5. Select or clear Mandatory.
  6. Select Save.

[Screenshot: Edit requirement row]

Expected result: The row updates and remains in the requirement list.

Step-By-Step: Mark A Low-Confidence Requirement Reviewed

  1. Locate a row marked Review required.
  2. Compare the extracted text to the source solicitation.
  3. If the row is correct, select Reviewed.
  4. If the row is not correct, edit it instead.
  5. Repeat until no low-confidence rows remain unreviewed.

[Screenshot: Review required row with Reviewed action]

Expected result: The row no longer blocks confirmation.

Step-By-Step: Add A Requirement Manually

  1. Select Add requirement manually.
  2. Enter the requirement text.
  3. Select a category.
  4. Select or clear Mandatory.
  5. Select Add.

[Screenshot: Add Requirement form]

Expected result: The new manual requirement is appended to the active requirement list.

Step-By-Step: Delete A False Positive

  1. Locate the false-positive or duplicate requirement.
  2. Select the delete control.
  3. Confirm the row is removed from the active list.

[Screenshot: Requirement row delete control]

Expected result: The requirement is excluded from the active list. Verified backend behavior soft-deletes requirements rather than hard-deleting them.

Step-By-Step: Confirm Requirements

  1. Confirm the document status is Completed.
  2. Confirm all rows marked Review required have been edited or marked reviewed.
  3. Confirm the requirement list is complete and accurate.
  4. Select Confirm Requirements.
  5. Wait for the success message.

[Screenshot: Confirm Requirements button]

Expected result: Pursuite displays Requirements confirmed! Compliance Matrix and Pipeline are now available. The requirement set is locked and downstream proposal workflows unlock.

Warning: After confirmation, the list is locked for downstream workflows. Re-processing can replace active rows only after a successful extraction run.

Step-By-Step: Retry Or Re-Process

  1. Open RFP Shredder.
  2. If processing failed, review the displayed error.
  3. If the failure is not a CUI block, select Retry Processing.
  4. If needed, select Upload Different File or Replace File and upload a corrected file.
  5. Wait for processing to complete.
  6. Review the updated requirement list before confirming.

[Screenshot: Processing failed state with Retry Processing]

Expected result: A new processing run is queued. Existing visible requirements remain available if the new run fails.

Wizard Shred Process

The verified guided process is the Proposal Kickoff wizard. It is launched from Guided Setup or from the proposal workspace Kickoff wizard button when available.

The wizard is server-gated. It cannot advance past RFP confirmation until the proposal has an uploaded, shredded, and confirmed RFP.

Verified Proposal Kickoff Wizard Steps

Wizard Step Verified Purpose
Proposal Basics Name the proposal and set top-level parameters such as proposal name and page limit.
RFP Ingestion Upload the solicitation PDF. The RFP Shredder extracts requirements automatically.
Review & Confirm Requirements Close the wizard, open the RFP Shredder panel, review and confirm requirements, then return to Guided Setup.
Compliance Matrix Preview Generate the matrix after requirements are confirmed.
Structure & Team Assign writers and reviewers from the proposal workspace.
Win Themes Optionally add win themes to steer drafting.
Summary & Launch Finish and mark the proposal draft-ready.

Note: The verified wizard upload step accepts PDF. The RFP Shredder panel itself accepts PDF and DOCX.

  1. Open the proposal workspace.
  2. Select Kickoff wizard if it is visible.
  3. If launching from the sidebar, select Guided Setup and start Proposal Kickoff: RFP -> Draft-Ready.
  4. Complete Proposal Basics.
  5. Continue to RFP Ingestion.

  4. Continue to Review & Confirm Requirements.

Troubleshooting

Issue Cause Resolution
Upload is rejected File is not PDF or DOCX. Upload a PDF or DOCX file.
Upload is rejected because of size File exceeds 50 MB. Reduce file size or split the solicitation according to internal policy.
RFP Shredder shows read-only access User is not Admin or Proposal Manager. Ask an Admin or Proposal Manager to upload, edit, or confirm.
Processing remains pending or processing Background extraction is still running. Wait for the panel to update automatically. If it remains stuck, ask an Admin to check worker and queue health.
Processing failed Extraction failed. Review the error, then use Retry Processing or upload a different file if available.
Processing failed with CUI_BLOCKED The document is classified as CUI and external CUI processing is not enabled. Ask an Admin to review CUI external-processing approval. Do not bypass CUI controls.
No requirements extracted The document may contain no readable text, or OCR may not have extracted text. Upload a higher-quality scan, or add requirements manually.
Confirm Requirements fails Low-confidence requirements require review. Edit each flagged row or mark it reviewed, then confirm again.
Compliance Matrix is unavailable RFP requirements have not been confirmed. Return to RFP Shredder and select Confirm Requirements.
Matrix generation fails with confirmation message A1 gate is not complete. Confirm requirements first.
Export fails Export generation or download failed. Retry the export and escalate if the issue persists.

Best Practices

  1. Upload the cleanest available solicitation document.
  2. Prefer text-searchable PDFs or DOCX files when available.
  3. Review low-confidence rows first.
  4. Compare requirement text against the source solicitation before confirming.
  5. Add missed requirements manually before confirmation.
  6. Delete false positives rather than leaving them for downstream compliance checks.
  7. Confirm only after the proposal team agrees the requirement set is usable.
  8. Generate the Compliance Matrix after requirements are confirmed and after meaningful proposal section content exists.
  9. Re-run or refresh the matrix after major proposal edits.
  10. Preserve audit context for CUI, AI processing, and confirmation decisions.

Audit Considerations

Verified audit-relevant behaviors include:

  • OCR fallback records an rfp.ocr_fallback audit event.
  • RFP upload, shredding, confirmation, and requirement edits are listed as common audited actions in admin documentation.
  • Low-confidence review gates prevent confirmation until uncertain rows are edited or marked reviewed.
  • CUI external-processing gates prevent classified RFP text from being sent to an AI provider unless approved.
  • Soft-delete behavior preserves history for removed requirements.

Warning: Do not route CUI or export-controlled solicitation material outside approved Pursuite workflows. If a CUI block appears, escalate to an Admin or security owner.
