Administration Overview

Purpose

This guide explains how administrators govern the Pursuite RFP Shredder and related downstream controls. It focuses on verified administration tasks: access control, AI provider readiness, CUI processing gates, processing status, troubleshooting, audit evidence, and backup considerations.

Administrator Responsibilities

Administrators are responsible for:

  • Maintaining user roles and access boundaries.
  • Ensuring at least one Proposal Manager or Admin can operate each proposal workflow.
  • Configuring and testing the AI provider required by RFP Shredder and Compliance Matrix workflows.
  • Governing CUI and external AI processing approvals.
  • Monitoring failed uploads, failed processing jobs, and blocked confirmation gates.
  • Preserving audit history for RFP upload, processing, edits, confirmation, CUI access, and AI activity.
  • Supporting backup and restore for the standalone deployment.

Security Model

The verified RFP permission family is:

Permission Roles
rfp:upload Admin, Proposal Manager
rfp:read Admin, Proposal Manager, Writer, Executive Approver
rfp:write Admin, Proposal Manager
rfp:confirm Admin, Proposal Manager

Related permissions include:

Permission Use
cui:read Read CUI-classified objects according to access controls.
cui:write Apply or update CUI classifications.
cui:admin Administer CUI controls, including classification removal where applicable.
compliance_matrix:read Read Compliance Matrix data.
compliance_matrix:write Generate or override Compliance Matrix data.

Architecture Overview

Verified RFP Shredder architecture:

Web proposal workspace
  -> RFP Shredder panel
  -> API RFP upload endpoint
  -> Storage
  -> RFP Shredder queue and worker
  -> Text extraction / OCR fallback / DOCX extraction
  -> AI requirement extraction
  -> rfp_documents and rfp_shredder_requirements data
  -> Confirm Requirements gate
  -> Compliance Matrix and Autonomous Pipeline unlock

Installation

Supported Deployment For Standalone 1.0

Existing administrator documentation verifies Pursuite 1.0 as a standalone SMB pilot. The supported production path for that release is the signed Windows installer or signed portable launcher running a local Docker-managed stack through Pursuite Manager.

Windows Installation Prerequisites

Requirement Verified Detail
Operating system Windows 10 or Windows 11.
Container runtime Docker Desktop current stable release.
Installer path Signed Pursuite installer or signed portable launcher.
Manager Pursuite Manager starts/stops the local stack, opens the local web app, and exposes logs, backups, restore, and diagnostics.

Step-By-Step: Launch Standalone Pursuite

  1. Install Docker Desktop current stable release on Windows 10 or Windows 11.
  2. Start Docker Desktop and wait until it reports ready.
  3. Run Pursuite-Standalone-Setup-1.0.0.exe or Pursuite-Standalone-Launcher-1.0.0.exe.
  4. Open Pursuite Manager.
  5. Confirm the Dashboard shows Standalone 1.0 Pilot, version v1.0.0, and local paths for data, environment, backups, launcher log, and bundled images.
  6. Select Launch Pursuite.
  7. Wait for the stack to become healthy.
  8. Sign in with the first-run Admin account.

[Screenshot: Pursuite Manager Dashboard]

Expected result: The local Pursuite web app opens and the Admin can sign in.

First-Run Admin

On a clean standalone install, Pursuite Manager creates %APPDATA%\Pursuite\first-run-admin.txt after the stack becomes healthy when the default organization has no Admin user. Use organization default, email admin@pursuite.local, and the temporary password in that file.

Warning: Rotate the temporary Admin password immediately by creating a named Admin user or resetting the local fallback password.

CUI External Processing Gate

When an RFP document is classified as CUI, the RFP Shredder worker checks whether the organization is approved for external CUI processing before sending document text to an AI provider.

Do not enable external CUI processing unless:

  1. The AI provider is approved for the data type.
  2. Contract terms permit that processing.
  3. Security and compliance owners approve the configuration.
  4. The approval is documented.

Warning: Do not bypass the CUI_BLOCKED state by moving controlled content outside Pursuite. Resolve the policy/configuration issue first.

  1. Confirm the restore request is approved.
  2. Notify impacted users.
  3. Stop active editing work.
  4. Open Pursuite Manager.

Troubleshooting

RFP Upload Or Processing Failures

Issue Cause Resolution
Upload rejected Unsupported file type. Use PDF or DOCX.
Upload rejected File exceeds 50 MB. Reduce file size or obtain a smaller source document.
User cannot upload User lacks Admin or Proposal Manager role. Update role only if justified, or have an authorized user upload.
Processing failed AI provider not configured or unavailable. Test AI Provider configuration.
Processing failed Worker or queue issue. Review Manager logs and queue/worker health.
Processing failed with CUI_BLOCKED CUI external processing is not approved. Review CUI policy and AI provider approval.
No requirements extracted PDF may be unreadable even after OCR. Ask the user to upload a higher-quality scan or add requirements manually.

Authentication Issues

Issue Cause Resolution
User cannot sign in Account inactive, wrong tenant, SSO issue, or MFA issue. Verify account status, tenant, SSO health, and MFA enrollment.
Admin item missing User is not an Admin. Confirm the role assignment.
MFA reset needed User lost authenticator. Verify identity and reset MFA according to policy.

Database Or Storage Issues

Issue Cause Resolution
Uploaded files missing Restore, storage issue, or wrong environment. Stop editing and check backup/restore status before retrying imports.
RFP document not found No document uploaded or document lookup returns 404. Upload an RFP to the proposal workspace.
Requirements disappear after re-process Successful re-process replaced old active rows. Review audit/history if needed. If re-process failed, old visible rows should remain.

Reporting Failures

Issue Cause Resolution
Compliance Matrix unavailable RFP requirements are not confirmed. Confirm requirements first.
Matrix generation fails AI provider, queue, or prerequisite issue. Verify AI provider, worker health, and RFP confirmation.
CSV export fails Export request or download failed. Retry and review API/log output if persistent.
DOCX/PDF export unavailable Current UI availability not verified. [SME Review Required: Confirm release-specific export controls.]
