1. Overview

This reference documents every REST API endpoint available in Pursuite. It is intended for third-party developers building integrations that require full account management, proposal generation, and compliance interoperability.
Base URL https://your-instance.pursuite.com/api/v1
Authentication Authorization: Bearer <accessToken> on all protected routes
Content-Type application/json (all requests unless noted)
Success envelope { "data": <payload> }
Error envelope { "error": "...", "message": "...", "statusCode": 400 }

Permission model: each endpoint requires a specific permission. The admin system role grants all permissions. Narrower roles grant subsets — see Section 2.

3. Authentication

Method Endpoint Auth Description
POST /auth/local Public Obtain token · body: { email, password, orgSlug } · returns { accessToken, expiresIn } + sets pursuite_refresh httpOnly cookie (30-day TTL)
POST /auth/refresh Cookie Renew token · no body · reads pursuite_refresh cookie · returns { accessToken, expiresIn }
GET /auth/login?org=<slug> Public SSO — redirects user to configured IdP
GET /auth/callback Public SSO return — issues token pair on success
POST /auth/logout Bearer Revoke session and clear refresh cookie
POST /auth/accept-invite Public Accept invitation · body: { token, password } · returns { accessToken, expiresIn }

Token Details

  • Access token TTL: 8 hours — pass as Authorization: Bearer <token>
  • Refresh token TTL: 30 days — httpOnly cookie pursuite_refresh scoped to /api/v1/auth/refresh
  • JWT payload: { sub, org, role, roles[], email, iat, exp }

Signature Verification

// Node.js
const crypto = require('crypto');
function verify(secret, rawBody, sigHeader) {
  const expected = 'sha256=' + crypto
    .createHmac('sha256', secret).update(rawBody).digest('hex');
  return crypto.timingSafeEqual(Buffer.from(expected), Buffer.from(sigHeader));
}

# Python
import hmac, hashlib
def verify(secret, raw_body, sig_header):
    expected = 'sha256=' + hmac.new(
        secret.encode(), raw_body, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, sig_header)

15. RFP Shredder (A1)

The RFP Shredder extracts requirements from an uploaded RFP PDF or DOCX using AI and produces a structured, editable requirement list. Confirming requirements unlocks the Compliance Matrix (A2) and Autonomous Pipeline (A3) for the proposal.

BUG-08 (v2.0): GET /proposals/:id/rfp-document previously returned 500 when no document had been uploaded. This is now a structured 404.

15.1 Document Upload & Status

Current implementation supports PDF and DOCX uploads up to 50 MB. RFP endpoints use the dedicated rfp:* permission family: rfp:upload, rfp:read, rfp:write, and rfp:confirm. 20. Deltek ProPricer & CPQ

Connects Pursuite to a Deltek ProPricer server via the ProPricer Bridge microservice — a .NET 8 sidecar that wraps the ProPricer 9.5 SDK (WebSocket/.NET protocol) and exposes a REST+JSON interface.

Architecture:

Pursuite API  ──(REST + JSON + X-Bridge-Key)──▶  ProPricer Bridge (.NET 8)
                                                           │
                                               (ProPricer SDK / WebSocket)
                                                           │
                                                           ▼
                                               ProPricer 9.5 App Server

Security model: ProPricer credentials (username, password, license key) reside only in the bridge's appsettings.json. Pursuite stores only the bridge URL and an AES-256-GCM encrypted bridge API key. Neither credential is ever returned in API responses.

Mock mode: the bridge runs in MockMode=true by default, returning realistic canned data — no ProPricer license required for development or testing. Set Bridge:MockMode=false and supply SDK DLLs for real connectivity.

21. Error Reference

HTTP Status Meaning When It Occurs
200 OK Request succeeded
201 Created Resource created successfully
202 Accepted Async job queued (assembly, compliance scan, pipeline run, export)
204 No Content Delete or action succeeded with no body
207 Multi-Status Batch import with mixed results — check errors[] array
400 Bad Request Validation failed — check details for field-level errors; bridge not configured
401 Unauthorized Missing or expired access token
403 Forbidden Authenticated but role lacks required permission (e.g. writer cannot confirm RFP requirements)
404 Not Found Resource does not exist or belongs to a different org; no RFP document uploaded (A1); proposal not found in ProPricer (bridge)
409 Conflict Duplicate detected (e.g. solicitation number); A1 not confirmed before A2 generation
413 Payload Too Large File upload exceeds 50 MB limit
415 Unsupported Media Type Non-PDF file uploaded to rfp-upload endpoint
422 Unprocessable Semantic error (e.g. section has content, cannot delete; work-share sum would exceed 100%)
423 Locked Proposal section is locked by another user
502 Bad Gateway Bridge microservice unreachable or returned an error
503 Service Unavailable AI provider not configured or external service unreachable
