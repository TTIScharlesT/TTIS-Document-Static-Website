1. Welcome To TTIS VBS

The TTIS AI Virtual Business System is the internal dashboard and automation platform for Thompson Telecom Infrastructure Solutions. It helps teams manage field operations, timecards, incidents, schedules, travel, finance, payroll, contracts, documents, Knowledge search, training, workflows, approvals, and AI-assisted work.

The system is role-aware. You will only see and use the tools your account is allowed to access.

1.1 Key Ideas

Term Meaning
Dashboard The browser interface for daily work.
VBS The backend system that powers workflows, approvals, and records.
ABAC Attribute-based access control; your role, division, auditor status, and task context determine access.
Gate A human approval point in a workflow.
Step-up A short second-factor verification used before sensitive actions.
Knowledge Searchable internal document and regulation corpus.
LYNNE CEO-facing executive assistant.
OPS assistants Operations-facing assistant surfaces for field support and management.

3. Sign In And Session Basics

3.1 Sign In

  1. Open the TTIS Dashboard URL provided by your administrator.
  2. Enter your username.
  3. Enter your password.
  4. Select Sign In.
  5. Wait for the dashboard to load your role-specific home view.

Expected result: your Overview page opens and the navigation reflects your permissions.

3.2 Change Your Password

  1. Open your account or security menu.
  2. Select Change Password.
  3. Enter your current password.
  4. Enter your new password.
  5. Confirm the new password.
  6. Save.

Expected result: future sign-ins use the new password.

3.3 Session Timeout

For security, sessions expire after inactivity. If your session expires:

  1. Save any work if the page allows it.
  2. Sign in again.
  3. Return to the task.

If you were performing a sensitive approval, you may also need to repeat step-up verification.

3.4 Sign Out

  1. Open your account menu.
  2. Select Sign Out.
  3. Close the browser tab if using a shared workstation.

Always sign out from shared or field devices.

4. Set Up MFA And Step-Up Verification

MFA protects sensitive actions. Even if you can sign in, some approvals or financial actions require a fresh step-up code.

4.1 Enroll An Authenticator App

  1. Sign in.
  2. Open Security or MFA Setup.
  3. Select Enroll TOTP or Set Up Authenticator.
  4. Open your authenticator app on your phone.
  5. Scan the QR code or enter the setup key.
  6. Save backup codes in a secure place.
  7. Enter the current 6-digit code.
  8. Submit.

Expected result: the dashboard confirms MFA enrollment.

4.2 Complete A Step-Up Prompt

When prompted:

  1. Open your authenticator app.
  2. Enter the current 6-digit code.
  3. Select Verify.
  4. Return to the action and submit again if needed.

Step-up assertions are short-lived. If you wait too long, the system may ask again.

4.3 Use A Backup Code

Use a backup code only if your authenticator app is unavailable.

  1. Choose backup-code verification if offered.
  2. Enter one unused backup code.
  3. Mark that code as used in your secure record.
  4. Contact an administrator if you are running out of backup codes.

  5. Confirm the alert state updates or the next owner is notified.

7. Use Knowledge Search

Knowledge Search helps you find internal documents, policies, regulations, and reference material.

7.1 Search

  1. Open Knowledge.
  2. Enter a question or phrase.
  3. Select Search.
  4. Review results and citations.
  5. Open a section result to inspect context.

Good searches:

  • "travel reimbursement policy"
  • "timecard approval deadline"
  • "DFARS cybersecurity clause"
  • "OSHA trenching requirements"
  • "proposal color review steps"

7.2 Review Results Safely

  1. Read the cited source.
  2. Confirm date and context.
  3. Do not rely on snippets alone for policy decisions.
  4. If the result affects finance, payroll, safety, HR, or contract compliance, confirm with the responsible owner.

7.3 When Search Returns No Results

Try:

  • fewer words
  • exact phrase from the document
  • alternate name for the policy
  • related regulation number

If the document should exist, contact an administrator to confirm ingestion.

8. Use Ask-LYNNE And OPS Assistants

Assistant access is role-scoped.

8.1 Ask-LYNNE

Ask-LYNNE is CEO-scoped. It can help with executive summaries, project context, Knowledge-grounded answers, calendar/workflow proposals, research, and structured outputs.

Steps:

  1. Open Ask-LYNNE.
  2. Type a clear request.
  3. Review sources before acting.
  4. If LYNNE proposes an action, review the action card.
  5. Confirm only if the action is correct.
  6. Complete step-up if prompted.

Example prompts:

  • "Summarize open gates by risk."
  • "Draft a status update for active VA opportunities."
  • "Find policy references for travel reimbursement."
  • "Prepare a project kickoff checklist for this award."

8.2 OPS Assistants

OPS assistants support operations-facing requests.

Steps:

  1. Open the operations assistant available to your role.
  2. Ask a specific operational question.
  3. Review grounded context and suggested next steps.
  4. Use normal dashboard forms for final submissions and approvals.

8.3 Assistant Safety Rules

  • Treat AI answers as decision support, not automatic approval.
  • Check citations.
  • Do not paste secrets, passwords, private keys, or unnecessary personal data.
  • Confirm action cards carefully.
  • If the answer conflicts with policy, follow policy and escalate.

9. Field Crew Workflows

9.1 View Your Schedule

  1. Open Schedule or Field Ops > Schedule.
  2. Review assigned shifts and project locations.
  3. Open a shift for details.
  4. Contact your manager if anything is missing or incorrect.

9.2 Submit A Timecard

  1. Open Timecards.
  2. Choose the current pay period.
  3. Add an entry for each work date.
  4. Enter hours.
  5. Select the correct charge code.
  6. Select labor class.
  7. Add a note if needed.
  8. Save as draft if still editing.
  9. Review totals.
  10. Check the attestation statement.
  11. Submit.

Expected result: the timecard moves to submitted status for manager review.

9.3 Correct A Timecard

  1. Open the timecard.
  2. Check status.
  3. If returned, read the manager or finance note.
  4. Edit the requested entries.
  5. Add a correction reason.
  6. Resubmit.

If the timecard is locked or already approved, contact your manager.

9.4 Submit A Daily Log

  1. Open Daily Logs.
  2. Select project.
  3. Select date.
  4. Enter crew count.
  5. Describe work performed.
  6. Add metrics such as footage, connections, or tests if applicable.
  7. Attach or reference supporting information if available.
  8. Submit.

Expected result: the daily log is recorded for project reporting.

9.5 Report An Incident Or Hazard

  1. Open Incidents.
  2. Select incident type: - injury - near miss - property damage - hazard
  3. Enter location.
  4. Enter severity.
  5. Describe what happened.
  6. Add body part if injury-related.
  7. Submit immediately.
  8. Notify your supervisor according to safety policy.

Do not wait until the end of the day for safety incidents.

9.6 Submit A Travel Request

  1. Open Travel.
  2. Select New Request.
  3. Enter destination.
  4. Enter dates.
  5. Enter business purpose.
  6. Enter traveler count.
  7. Submit.
  8. Review options when they are provided.
  9. Select the preferred option if your role allows.

Expected result: the travel request moves through review and booking handoff.

14. Approvals, Gates, And Step-Up Prompts

14.1 What A Gate Means

A gate is a human decision point. It may pause a workflow until an authorized person reviews evidence and decides.

14.2 Review A Gate

  1. Open Gates or click a pending gate from Overview.
  2. Read the title, workflow, age, and description.
  3. Review details and source records.
  4. Choose the appropriate decision: - approve - reject - request changes, if available
  5. Add a note.
  6. Complete step-up if prompted.
  7. Submit.

Expected result: the workflow advances, stops, or returns for changes.

14.3 If You Are Denied

If you receive 403 or an access-denied message:

  1. Confirm you are signed in as the correct user.
  2. Confirm the action belongs to your role.
  3. Confirm the record is in your division, if applicable.
  4. Contact your manager or administrator if access appears wrong.

Do not ask another user to perform actions under your name.

15. Notifications, Errors, And Access Denied Messages

15.1 Common Messages

Message Meaning What to do
Invalid credentials Username or password was wrong, or account is disabled Retry once, then contact admin.
Session expired You were inactive too long Sign in again.
Step-up required Sensitive action requires MFA Verify TOTP and retry.
Step-up enrollment required You need MFA before this action Enroll authenticator.
Access denied Role, division, or auditor posture does not allow action Contact manager/admin if unexpected.
Conflict or version mismatch Another edit changed the record Refresh and review before retrying.
Duplicate request Idempotency or replay protection detected a repeat Check whether the first submission succeeded.

15.2 When To Escalate

Escalate immediately if:

  • you see data for the wrong division or person
  • payroll or finance totals look wrong
  • an incident severity is incorrect
  • a gate is blocking urgent work
  • AI output cites missing or inappropriate sources
  • your device or credentials may be compromised

16. Safe Use Of AI Outputs

AI assists with search, drafting, summarization, workflow preparation, and operational support. You remain responsible for decisions you submit.

16.1 Do

  • Read citations.
  • Verify amounts, dates, names, contract numbers, and compliance claims.
  • Use action cards only after review.
  • Keep sensitive data inside approved dashboard workflows.
  • Ask a clearer follow-up if the answer is vague.

16.2 Do Not

  • Paste passwords, tokens, private keys, or MFA codes into chat.
  • Treat AI output as final legal, payroll, safety, or contract approval.
  • Submit an action card you do not understand.
  • Use another user's account.
  • Export sensitive data outside approved channels.

18. Getting Help

Contact your manager or system administrator when:

  • you cannot sign in
  • MFA is lost
  • your role or division is wrong
  • a record is missing
  • a workflow is stuck
  • an approval is urgent
  • you see data you should not see
  • you suspect a security issue

When reporting a problem, include:

  • what you were trying to do
  • page or workflow name
  • approximate time
  • error message
  • record ID, if available
  • screenshot without secrets or private key material
