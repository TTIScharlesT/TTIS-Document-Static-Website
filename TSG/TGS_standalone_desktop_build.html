TSG Platform Standalone Desktop Build

This guide describes the portable Windows desktop package for running TSG Platform as a local standalone service. The desktop build is intended for demos, operator workstations, isolated lab installs, and customer-support environments that need the admin and portal UI without a cloud deployment.

Included files

  • desktop/TSG-Platform-Launcher.ps1
  • desktop/setup/Install-TSG-Platform-Desktop.ps1
  • Application source under api, core, database, seed_data, and scripts
  • requirements.txt
  • Product and deployment documentation in Markdown and HTML
  • desktop-manifest.json

The launcher creates a local Python virtual environment, installs dependencies, initializes a SQLite database, starts Uvicorn on 127.0.0.1:8010, and opens the admin console.

Build the package

powershell -ExecutionPolicy Bypass -File scripts\build_desktop_standalone.ps1

The default output is:

dist\desktop\TSG-Platform-Desktop-0.2.0-rc.1.zip
dist\desktop\TSG-Platform-Desktop-0.2.0-rc.1.zip.sha256

Use -SkipTests only for a quick local packaging check:

powershell -ExecutionPolicy Bypass -File scripts\build_desktop_standalone.ps1 -SkipTests

Install on Windows

  1. Extract the ZIP package.
  2. Run the installer:
powershell -ExecutionPolicy Bypass -File desktop\setup\Install-TSG-Platform-Desktop.ps1 -CreateDesktopShortcut

The default install location is:

%LOCALAPPDATA%\Programs\TSGPlatform

The installer creates a Start Menu shortcut and, when requested, a desktop shortcut.

Launch manually

powershell -ExecutionPolicy Bypass -File desktop\TSG-Platform-Launcher.ps1

Useful options:

powershell -ExecutionPolicy Bypass -File desktop\TSG-Platform-Launcher.ps1 -Port 8020
powershell -ExecutionPolicy Bypass -File desktop\TSG-Platform-Launcher.ps1 -NoBrowser
powershell -ExecutionPolicy Bypass -File desktop\TSG-Platform-Launcher.ps1 -DataDir "$env:USERPROFILE\TSGPlatformData"

Runtime data

By default, the launcher stores data under:

%LOCALAPPDATA%\TSGPlatform

The SQLite database is:

%LOCALAPPDATA%\TSGPlatform\tsg_platform.db

The launcher sets these local defaults when they are not already present:

TSG_ENV=development
TSG_DB_BACKEND=sqlite
TSG_DB_PATH=<DataDir>\tsg_platform.db
TSG_ALLOWED_HOSTS=127.0.0.1,localhost

For a custom environment file, pass -EnvFile path\to\.env.local. Values in that file are loaded before startup.

Verify the desktop service

Invoke-RestMethod http://127.0.0.1:8010/health
Invoke-RestMethod http://127.0.0.1:8010/ready

Open the admin console:

http://127.0.0.1:8010/admin

Development credentials are shown on the login page only when TSG_ENV is not production.

Operational notes

  • The desktop build uses SQLite and is not a public internet deployment.
  • Keep the generated .zip.sha256 file with the package for transfer validation.
  • If dependency installation is pre-provisioned, use -SkipDependencyInstall.
  • Stop the service by closing the PowerShell window that is running the launcher.
  • Back up the SQLite database before replacing or moving the workstation install.
